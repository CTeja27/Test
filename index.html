How it works (Host + Client)

On first launch, choose whether a computer is the Host or a Client. Clients can scan the LAN to auto-fill the Host server URL, or enter it manually.

1

Choose Host or Client

Host runs the backend locally (Docker). Client connects to the Host URL and shares the same data.

System requirements

Recommended for smooth Host + Client deployments.

Host machine

Docker Desktop

Only the Host typically needs Docker to run backend services locally.

Network

Same Wi-Fi/LAN

Clients must be on the same network as the Host for auto-discovery to work.

Ports

Open Host ports

Allow UDP 49501 and TCP 4000/5001 on the Host firewall.
