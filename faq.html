What is AMS Desktop?

AMS Desktop is a local-first management workspace inspired by AMS Network. It’s built for arts organizations that need a clean system to manage members, projects, campaigns, billing, reporting, and media workflows — while keeping the team on one shared source of truth.

One computer becomes the Host (runs the backend services locally), and other computers join as Clients on the same Wi-Fi/LAN to use the same shared data in real time.

Product overview
What is AMS Desktop used for? ⌄
AMS Desktop helps an organization manage operational work in one system:
  • Membership — organization contacts / members
  • Projects — track work and collaboration
  • Campaigns + Applications — intake + workflow tracking
  • Programs + Submissions — structured entries and statuses
  • Billing + Payments — invoice items and payment status
  • Reporting — summaries for organization decisions
  • Donations — donor records and visibility
  • MediaConvert — start/monitor transcoding jobs (Host runs the service)
The goal: your team stops tracking everything in scattered docs/spreadsheets and works from one shared workspace.
Who is it for? ⌄
AMS Desktop is designed for arts organizations and small teams that want:
  • A clean internal tool they can run locally
  • Shared access for multiple computers in the same office/network
  • Simple onboarding: Host once, then Clients join
Is AMS Desktop cloud-based? ⌄
AMS Desktop is local-first. It’s meant for teams working on the same network (Wi-Fi/LAN). Remote access outside the office would need additional networking/VPN setup.
Where is the data stored? ⌄
Your shared data lives on the Host machine because the backend services run there. Clients connect to the Host to read/write the same shared data.
Host + Client
What’s the difference between Host and Client? ⌄
  • Host = the “server computer.” It runs the backend services and provides the workspace.
  • Client = any other computer. It connects to the Host and uses the same data.
Think of it like: one Host powers the system, Clients are the workstations.
Do all computers need Docker? ⌄
Usually only the Host needs Docker Desktop because it runs the services locally. Clients typically just need AMS Desktop and access to the Host on the network.
What is “Find Server Automatically”? ⌄
It scans the local network to detect the Host and auto-fill the server URL. If it can’t detect the Host, you can connect manually using: http://HOST_IP:4000
Which ports are used? ⌄
Common ports on the Host:
  • UDP 49501 — LAN discovery (auto-find)
  • TCP 4000 — backend API
  • TCP 5001 — MediaConvert service
MediaConvert
What does MediaConvert do inside the app? ⌄
MediaConvert inside AMS Desktop lets your organization effortlesly transcode your videos thru AWS mediaconvert and save them in S3 buckets.
Why does it work best on the Host? ⌄
Because the Host is where backend services run. This keeps the workflow consistent: one machine executes jobs; the whole team sees the result.
Troubleshooting
“No server found” on Client — what should I check? ⌄
Checklist:
  • Host is running AMS Desktop in Host mode and services are up.
  • Host + Client are on the same Wi-Fi/LAN.
  • Host firewall allows UDP 49501 and TCP 4000 (and TCP 5001 if needed).
  • Try manual entry on Client: http://HOST_IP:4000
Client connects but data looks empty / not updating — why? ⌄
Most common causes:
  • Client is pointing to the wrong Host URL
  • Host IP changed after reboot/network change
  • Host services restarted and Client needs to reconnect
Fix: re-run auto-find or update the manual Host URL.
I changed Wi-Fi and Clients stopped working — what now? ⌄
When Wi-Fi changes, the Host IP can change. Re-run auto-find on Clients or manually update the Host URL.
How do I reset back to first-time setup? ⌄
Use the app’s Reset / Back to Setup option. It returns you to Host/Client selection and lets you reconfigure the server URL.
