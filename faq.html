What is AMS Desktop?

AMS Desktop is a desktop app where one computer can run as a Host (server) and other computers connect as Clients on the same Wi-Fi. Clients can automatically discover the Host and share the same real-time workspace. :contentReference[oaicite:17]{index=17} :contentReference[oaicite:18]{index=18}

What’s the difference between Host and Client?

Host runs backend services locally using Docker and creates the Organization + Admin during setup. Client connects to the Host’s server URL (manually or via auto-discovery). :contentReference[oaicite:19]{index=19}

Which ports need to be open?

For clients to find and connect to the Host, the Host machine should allow:

  • UDP 49501 — LAN discovery broadcast
  • TCP 4000 — Rails API service
  • TCP 5001 — MediaConvert service

Host mode can also attempt to set Windows Firewall rules automatically (with an Admin prompt). :contentReference[oaicite:20]{index=20} :contentReference[oaicite:21]{index=21}

Do all computers need Docker?

No. Typically only the Host needs Docker Desktop because it runs the backend services locally. Clients connect over the network to the Host’s server. :contentReference[oaicite:22]{index=22}

What services run on the Host?

The standard stack uses Docker Compose to run a Rails service on :4000, Postgres, Redis, and a MediaConvert service on :5001. :contentReference[oaicite:23]{index=23}

“No server found” when I click Find Server Automatically — what do I do?

Make sure:

  • The Host is running AMS Desktop in Host mode.
  • Both computers are on the same Wi-Fi.
  • Windows Firewall allows UDP 49501 and the service ports TCP 4000 / TCP 5001.
  • If needed, manually enter the Host URL in the client setup (example: http://HOST_IP:4000).

This matches the setup screen behavior for LAN scan + fallback manual URL entry. :contentReference[oaicite:24]{index=24}

How do I reset AMS Desktop back to the first-time setup?

On the Login screen, click “Reset / Back to Setup”. This clears host/client mode and server URL and returns you to setup. :contentReference[oaicite:25]{index=25}

How does MediaConvert work in AMS Desktop?

AMS Desktop can start jobs and check job status through the local MediaConvert service on http://127.0.0.1:5001 (Host side). :contentReference[oaicite:26]{index=26}
